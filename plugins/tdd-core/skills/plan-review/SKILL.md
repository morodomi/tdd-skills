# Plan Review

PLANフェーズの設計を3観点で並行レビューし、問題を早期発見する。

## Progress Checklist

コピーして進捗を追跡:

```
plan-review Progress:
- [ ] Cycle doc確認
- [ ] 並行レビュー実行（3観点）
- [ ] 結果統合
- [ ] 分岐判定（0件/1件以上）
```

## Workflow

### Step 1: Cycle doc確認

```bash
ls -t docs/cycles/*.md 2>/dev/null | head -1
```

PLANセクション（設計方針、Test List、変更予定ファイル）を読み込む。

### Step 2: 並行レビュー実行

3観点で並行レビュー:

| 観点 | チェック内容 |
|------|------------|
| **設計整合性** | Test Listのカバレッジ、変更ファイルの妥当性、依存関係 |
| **セキュリティ設計** | 認証・認可、入力バリデーション、機密データ |
| **バージョン互換性** | 最新バージョン確認、非推奨API、破壊的変更 |

### Step 3: 結果統合

Critical/Important/Optionalで分類して出力。

### Step 4: 分岐判定

#### 0件の場合
→ REDフェーズへ自動進行

#### 1件以上の場合

```
================================================================================
推奨: PLAN に戻って修正

選択してください:
1. PLAN に戻る（推奨）
2. RED に進む（問題を認識した上で）
================================================================================
```
