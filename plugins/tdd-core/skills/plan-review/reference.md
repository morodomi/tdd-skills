# Plan Review - Reference

## 詳細なレビュー観点

### 観点1: 設計整合性

- Test Listがスコープをカバーしているか
- 変更予定ファイルが妥当か
- 依存関係に見落としがないか

### 観点2: セキュリティ設計

- 認証・認可の考慮があるか
- 入力バリデーションの設計があるか
- 機密データの扱いが適切か

### 観点3: バージョン互換性

- Environmentのバージョンが最新か
- 使用予定のAPIが非推奨でないか
- 破壊的変更の影響がないか

## 出力形式

問題を発見した場合:

```markdown
### [問題タイトル]
- **観点**: 設計整合性 / セキュリティ設計 / バージョン互換性
- **重要度**: Critical / Important / Optional
- **問題**: 具体的な問題の説明
- **推奨対応**: 修正案
```

## 結果統合の出力例

```
================================================================================
plan-review 完了
================================================================================
問題: X件（Critical: A, Important: B, Optional: C）
```

## 問題なしの場合

```
================================================================================

================================================================================

================================================================================
```
